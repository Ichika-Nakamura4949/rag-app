# 要件定義

## 機能要件

### 1. ドキュメントアップロード

| 項目 | 内容 |
|---|---|
| 対応形式 | PDF, DOCX |
| アップロード方法 | Webブラウザからのファイル選択 |
| ファイルサイズ上限 | 50MB / ファイル |
| 処理内容 | テキスト抽出 → チャンク分割 → Embedding生成 → ベクトルDB保存 |
| フィードバック | アップロード進捗・処理状況の表示 |

### 2. チャット（質問応答）

| 項目 | 内容 |
|---|---|
| 入力 | テキストによる自然言語の質問 |
| 出力 | アップロード済みドキュメントの内容に基づく回答 |
| 参照元表示 | 回答の根拠となったドキュメント名・該当箇所の表示 |
| 会話履歴 | セッション内での会話履歴保持 |

### 3. ドキュメント管理

| 項目 | 内容 |
|---|---|
| 一覧表示 | アップロード済みドキュメントの一覧を表示 |
| 削除 | ドキュメントの削除（ベクトルDBからも削除） |
| メタデータ | ファイル名、アップロード日時、ファイルサイズの表示 |

## 非機能要件

### パフォーマンス

- チャット応答: 10秒以内（LLM API呼び出し含む）
- ドキュメントアップロード: 50ページ程度のPDFを30秒以内に処理
- ベクトル検索: 1秒以内

### セキュリティ

- APIキーはサーバーサイドで管理し、フロントエンドに露出しない
- アップロードファイルのバリデーション（形式・サイズチェック）
- 環境変数による秘密情報管理

### 可用性

- ローカル開発環境での動作を前提とする（初期フェーズ）
- ChromaDBのデータ永続化（アプリ再起動でデータが消えない）

### 保守性

- バックエンドとフロントエンドを分離した構成
- 型ヒント（Python）/ strict mode（TypeScript）による型安全性
- 自動APIドキュメント（FastAPI Swagger UI）

## 画面構成

### チャット画面（メイン）

- チャットメッセージ一覧（ユーザー質問 + AI回答）
- テキスト入力欄 + 送信ボタン
- 回答に含まれる参照元ドキュメントの表示

### ドキュメント管理画面

- アップロード済みドキュメント一覧
- ファイルアップロードエリア（ドラッグ＆ドロップ対応）
- ドキュメント削除ボタン

## 将来の拡張案

- **Slack連携**: Slackボットとして質問応答を提供
- **Notion連携**: NotionページをRAGのソースとして取り込み
- **認証機能**: ユーザー認証・アクセス制御の追加
- **マルチテナント**: チーム/部門ごとにドキュメントスペースを分離
- **ストリーミング応答**: SSEによるリアルタイムなトークン出力
- **会話履歴の永続化**: DB保存によるセッション跨ぎの履歴管理
- **ベクトルDB移行**: Pinecone/Qdrant等のマネージドサービスへの移行
- **ドキュメント形式の拡張**: Markdown, HTML, PowerPoint等への対応
